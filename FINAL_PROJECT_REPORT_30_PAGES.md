# SECTION 1: INTRODUCTION

## 1.1 Project Overview

The AI-Powered Grading System is a comprehensive, full-stack web application designed to revolutionize the educational assessment process for programming courses. By leveraging cutting-edge artificial intelligence, advanced algorithms, and modern web technologies, the system provides automated, accurate, and detailed feedback on student code submissions.

### Vision Statement
To create an intelligent, scalable, and user-friendly platform that empowers educators to focus on teaching while providing students with instant, actionable feedback on their programming assignments.

### Mission Statement
To democratize quality education by making advanced grading technology accessible to educational institutions worldwide, regardless of size or budget.

### Core Values
- **Innovation:** Leveraging latest AI and technology advancements
- **Quality:** Maintaining highest standards in code and output
- **Accessibility:** Ensuring platform is usable by all
- **Security:** Protecting user data and institutional integrity
- **Scalability:** Supporting growth from classrooms to universities

## 1.2 Problem Statement

### Traditional Grading Challenges

#### 1. Time Consumption
- Manual code review takes 15-30 minutes per submission
- Lecturers spend 60-80% of time on grading
- Feedback delay of 3-7 days affects learning
- Burnout and reduced teaching quality

#### 2. Inconsistency
- Different grading standards among evaluators
- Subjective assessment variations
- Fatigue-induced errors
- Bias in manual evaluation

#### 3. Limited Feedback
- Generic comments without detailed insights
- No immediate feedback for students
- Lack of personalized improvement suggestions
- Insufficient code quality analysis

#### 4. Plagiarism Detection
- Manual detection is time-intensive
- Basic tools miss sophisticated copying
- Cross-language plagiarism undetected
- Difficult to prove without evidence

#### 5. Scalability Issues
- Class size limitations (typically 30-50 students)
- Cannot handle multiple assignments simultaneously
- No real-time monitoring capabilities
- Resource-intensive grading process

### Market Gap Analysis

**Current Solutions:**
- Basic auto-graders: Limited to test case execution
- Commercial platforms: Expensive, closed-source
- Open-source tools: Lack AI features, poor UX
- Manual systems: Not scalable, time-consuming

**Our Solution:**
A comprehensive platform combining AI, automation, security, and user experience to address all identified gaps.

# SECTION 2: SYSTEM ARCHITECTURE

## 2.1 Technical Stack Overview

### Backend Technologies

#### Core Framework
- **Flask 2.3.3:** Lightweight Python web framework
- **Python 3.13.6:** Latest stable Python version
- **Werkzeug 2.3.7:** WSGI utility library

**Rationale:** Flask provides flexibility, excellent documentation, and a rich ecosystem of extensions while maintaining simplicity.

#### Database Layer
- **MongoDB 6.0:** NoSQL document database
- **PyMongo 4.5.0:** MongoDB Python driver
- **Flask-PyMongo 2.3.0:** Flask-MongoDB integration

**Rationale:** MongoDB's flexible schema supports rapid development and handles diverse data structures efficiently.

#### Authentication & Security
- **Flask-JWT-Extended 4.5.3:** JWT token management
- **Passlib 1.7.4:** Password hashing
- **PyOTP 2.9.0:** Two-factor authentication
- **Flask-Talisman 1.1.0:** Security headers
- **Flask-Limiter 3.5.0:** Rate limiting

**Rationale:** Industry-standard security libraries ensuring OWASP compliance.

#### AI/ML Integration
- **OpenAI 1.54.0:** GPT-4 API integration
- **Custom NLP Engine:** Code analysis algorithms
- **TF-IDF Vectorization:** Plagiarism detection

**Rationale:** Leverages state-of-the-art language models for intelligent code assessment.

#### Task Processing
- **Celery 5.3.4:** Distributed task queue
- **Redis 5.0.1:** Message broker and cache
- **Gunicorn 21.2.0:** Production WSGI server

**Rationale:** Enables asynchronous processing for time-intensive operations.

#### Communication
- **Flask-Mail 0.9.1:** Email integration
- **Flask-SocketIO 5.3.6:** WebSocket support

```python
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target: return mid
        elif arr[mid] < target: left = mid + 1
        else: right = mid - 1
    return -1
```

```python
    Time Complexity: O(nÂ²) comparisons
    Optimized with parallel processing
    """
    results = []
    total_comparisons = n * (n - 1) / 2
    
    # Use Celery for parallel processing
    tasks = []
    for i in range(len(submissions)):
        for j in range(i + 1, len(submissions)):
            task = compare_submissions.delay(
                submissions[i], 
                submissions[j]
            )
            tasks.append(task)
    
    # Collect results
    for task in tasks:
        result = task.get(timeout=300)
        if result['similarity'] >= 91:
            results.append(result)
    
    return results
```

```python
    results = []
    
    for test_case in test_cases:
        # 1. Create temporary directory
        temp_dir = create_temp_directory()
        
        # 2. Write code to file
        code_file = write_code_file(temp_dir, code, language)
        
        # 3. Compile if necessary
        if language in ['java', 'cpp', 'c']:
            compile_result = compile_code(code_file, language)
            if not compile_result.success:
                return CompilationError(compile_result.errors)
        
        # 4. Execute code
        try:
            output = run_in_sandbox(
                code_file=code_file,
                input_data=test_case.input,
                timeout=30,
                memory_limit="256m"
            )
            
            # 5. Compare output
            passed = compare_output(
                output.stdout,
                test_case.expected_output
            )
            
            results.append({
                'test_case': test_case.id,
                'passed': passed,
                'output': output.stdout,
                'execution_time': output.time,
                'memory_used': output.memory
            })
            
        except TimeoutError:
            results.append({
                'test_case': test_case.id,
                'passed': False,
                'error': 'Execution timeout (30s)'
            })
        except MemoryError:
            results.append({
                'test_case': test_case.id,
                'passed': False,
                'error': 'Memory limit exceeded (256MB)'
            })
        finally:
            # 6. Cleanup
            cleanup_temp_directory(temp_dir)
```

```python
def flexible_match(actual, expected, tolerance=1e-6):
    try:
        actual_float = float(actual)
        expected_float = float(expected)
        return abs(actual_float - expected_float) < tolerance
    except ValueError:
        return actual.strip() == expected.strip()
```
